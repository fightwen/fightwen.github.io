我的第一篇github blog文心得

前言

Blog建立了，我希望這邊是很認真地講coding技術的地方，畢竟吸收的資訊多，忘的也很多，

新手需要更多磨練督處自己才行。

Github

光是只有push專案不夠，很多知識還是很貧乏，沒辦法很有口條的說清楚know how，

於是動起想建立github頁面的念頭。以後找技術就專注於尋找github即可。

Blog

在這個微網誌盛行的時代下，我發現我被養成”只會說一句話”的習慣，我需要練習很有條理地寫一篇文章，

還有，Blog形式有助於日後回顧，畢竟學習的過程是很可貴的，就這樣忘了很可惜。

我找了很多Blog服務，很少有對code有很好的支援，都必須加入外掛，我想直接用技術圈的方式，應該就可以跨過這個麻煩了(?)

Hexo

感謝這個框架讓github架起Blog變簡單了，意外發現是台灣人做的，剛好官網也有提供多語言，很佛心呢。
不過有些解說看不是很懂，感覺還是要懂技術的人比較容易建得起來。

最後感謝手把手教學
